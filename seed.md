## Section 6 — Growth Loop

One tick = **Orient → Plan → Execute → Verify → Record → Publish**

1. **check-lifecycle** — gate: grow / replant / consolidate / expand / dormant
2. **plan-roadmap** — select 1–3 items from Now/Backlog (content | structure | meta)
3. **Curator** (content items) or **build-structure** skill (structure items) — execute
4. **sync-seed** — refresh sections 1–7
5. **encode-seed** — append tick entry to section 8
6. **publish-session** — commit + push to main

Unattended: `.github/workflows/grow.yml` (scheduled).

## Section 7 — Rebuild Procedure

To rebuild this repo from scratch using only this file:

1. `git clone <repo>` into a fresh directory.
2. Ensure `.github/` framework files are present (copy from canonical source if needed).
3. Run `/genesis` (blank) — it reads `seed.md`'s Concept Definition and regenerates everything.
4. Run `/grow` to produce first real content.

To retarget to a new concept: change `concept.subject` and `concept.taxonomy`, delete content folders, run `/genesis "<new concept>"`.
